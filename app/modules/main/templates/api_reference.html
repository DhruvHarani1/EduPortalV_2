Getting Started

Welcome to the EduPortal Developer API. This reference documents the full list of available endpoints, corresponding to the application's internal routing table.

Base URL

https://eduportal.com

Response Codes

Code Description
200 OK Request succeeded.
302 Found Redirection (Common for Form POSTs).
401 Unauthorized Authentication required.
403 Forbidden Access denied for your role.
404 Not Found Resource does not exist.
500 Server Error Internal system failure.
Security

Authentication

Endpoints for establishing and terminating user sessions.

GET, POST /auth/login

Endpoint: auth.login

Main login interface. POST with email, password, and role to authenticate.

GET /auth/logout

Endpoint: auth.logout

Clears the session and redirects to index.

Module

Student API

Complete reference for student-facing endpoints.

GET /student/dashboard
student.dashboard

Aggregates all critical data for the student's landing page: attendance summaries, fee status, upcoming events, and recent notices.

Response Structure (JSON)

{ "student": { "name": "Alex Johnson", "course": "B.Tech Computer Science", "semester": 6 }, "attendance_pct": 85.5, "pending_fees": 0, "latest_spi": 9.2, "next_event": { "title": "Tech Symposium", "date": "2024-04-20" }, "notices": [ { "title": "Exam Schedule Released", "date": "2024-04-10" }, { "title": "Holiday on Friday", "date": "2024-04-08" } ] }
GET /student/attendance
student.attendance

Returns detailed subject-wise attendance statistics. Calculates recovery needed for subjects below 75%.

Response Structure (HTML View Logic)

  • overall_percent: Aggregate percentage across all subjects.
  • subjects: List of objects containing:
    • name: Subject Name
    • present: Days present
    • total: Total days taught
    • percent: (Present/Total)*100
    • recovery_needed: Number of consecutive classes needed to reach 75%.
GET /student/academics
student.academics

Displays academic history grouped by Exam Events. Calculates SPI for each exam and overall CGPI.

Key Data Points

Exams List

Sorted by date (latest first). Contains results for all subjects in that exam.

Grading Logic

AA (>90), AB (>80), BB (>70), etc. Points are calculated as per credit weightage.

GET /student/academics/marksheet/<id>
student.download_marksheet

Generates a printable Marksheet for a specific Exam Event ID.

Parameter Type Description
exam_id Integer (Path) Unique ID of the Exam Event (e.g., 5 for "Mid Term 2024").
GET /student/fees
student.fees

Lists all fee records split into two groups: Pending/Overdue and Paid History.

POST /student/fees/pay/<id>
student.pay_fee

Initiates payment for a specific fee record. Currently simulates a successful transaction updating the status to 'Paid'.

Response (JSON)

{ "status": "success", "message": "Payment successful!", "transaction_id": "TXN17150112347" }
POST /student/queries/create
student.create_query

Submit a new academic or general query to a faculty member.

Form Data Parameters

Field Type Required Description
subject_id Integer Yes ID of the subject interacting with.
title String Yes Brief subject line of the query.
content String Yes Detailed explanation of the issue.
image File No Optional screenshot/image attachment.
GET /student/timetable
student.timetable

Renders the weekly schedule. Uses heuristic matching if specific Timetable entries are missing (mapping days to subjects).

Module

Faculty API

Endpoints for faculty management (Attendance, Marks, Mentorship).

GET /faculty/dashboard
faculty.dashboard

Overview for faculty: today's schedule, pending attendance tasks, mentee count, and recent notices.

POST /faculty/attendance
faculty.attendance

Submit daily attendance for a specific subject and batch. Supports bulk updates.

Form Data Parameters

Field Type Description
subject_id Integer ID of the subject class.
date Date (YYYY-MM-DD) The date of the lecture.
status_{student_id} String Status for each student (e.g., `status_101=Present`, `status_102=Absent`).

Success Response

{ "status": "success", "message": "Attendance marked for 45 students.", "timestamp": "2024-05-20T10:30:00Z" }
POST /faculty/marks
faculty.marks

Submit marks for a specific Exam Paper. Validates against maximum marks defined in the paper.

Form Data Parameters

Field Description
exam_id Selected Exam Event ID (Query Param in GET).
paper_id Specific Exam Paper ID (Query Param in GET).
marks_{student_id} Numeric value for each student.
GET /faculty/classes
faculty.classes

Lists assigned subjects. Also processes Syllabus uploads via POST.

POST /faculty/notices
faculty.notices

Send a notice to a specific class or individual student.

Form Data Parameters

Field Required Description
title Yes Subject of the notice.
target_type Yes 'class' or 'individual'.
target_course Cond. Required if target_type is 'class'.

Admin API

System-wide management endpoints (Users, Courses, Exams, Reports).

POST /admin/students/add
admin.add_student

Create a new student profile and associated user account. Handles password hashing and initial setup.

Form Fields

Field Name Type Required Description
nameStringYesFull Name
emailEmailYesUnique Email Address
passwordStringYesInitial Password
enrollment_numberStringYesUnique ID
course_nameStringYese.g., "B.Tech"
semesterIntegerYesCurrent Semester
batch_yearStringNoe.g., "2024-2028"
date_of_birthDateNoYYYY-MM-DD
phone_numberStringNoContact Number
POST /admin/students/import
admin.import_students

Bulk upload students via CSV file.

CSV Specification

name,email,password,enrollment_number,course_name,semester John Doe,john@example.com,pass123,EN1001,B.Tech,1 Jane Smith,jane@example.com,pass456,EN1002,MCA,3
POST /admin/faculty/add
admin.add_faculty

Create a new faculty profile. allow assigning multiple subjects initially.

Key Parameters

designatione.g., "Professor", "Assistant Professor"
assigned_subjectsMulti-select list of Subject Names
photoBinary file upload (image)
POST /admin/timetable/generate
admin.generate_timetable

Triggers the automatic scheduling algorithm for a specific course and semester. Replaces any existing schedule.

Algorithm Inputs

  • Constraints: Days per week, Slots per day.
  • Subjects: Minimum weekly lectures per subject.
  • Logic: Randomizes subject pool, fills slots ensuring daily consistency, and respects faculty conflicts (if enabled).
POST /admin/exams/create
admin.create_exam_event

Initializes a new exam session (e.g., "End Sem May 2024"). Requires defining the start/end dates and the target course/semester.
