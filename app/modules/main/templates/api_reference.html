Overview

Getting Started

Welcome to the EduPortal Developer API. This reference documents both the JSON-based internal APIs used for asynchronous data fetching and the standard Form-based endpoints used for core operations.

Base URL

https://eduportal.com

Response Codes

Code Description
200 OK Request succeeded.
302 Found Redirection (Common for Form POSTs).
401 Unauthorized Authentication required.
403 Forbidden Access denied for your role.
404 Not Found Resource does not exist.
500 Server Error Internal system failure.
Security

Authentication

EduPortal uses session-based authentication. Clients must maintain cookies describing the session state.

POST /auth/login

Establish a session.

Form Data 
email: "user@example.com"
password: "your_password"
role: "student" | "faculty" | "admin"
Response

Returns 302 Redirect to the role-specific dashboard upon success.

GET /auth/logout

Terminate the current session.

Response

Returns 302 Redirect to the landing page.

Analytics API (JSON)

Reports & Intelligence

Endpoints for retrieving aggregated analytics. Returns application/json.

GET /api/reports/student-performance

Get deep insights into student academic performance, including "Academic DNA" (Radar), consistency scores, and growth velocity.

Response Example 
{
  "consistency": [
    {
      "growth": 12.5,
      "name": "Arjun Patel",
      "x": 78.5,
      "y": 4.2
    }
  ],
  "danger_zone": [
    {
      "avg": 35.0,
      "name": "Rohan Gupta",
      "risk": "Critical"
    }
  ],
  "insights": {
    "review": "The batch is currently Improving.",
    "status": "Improving",
    "suggestion": "Maintain current momentum.",
    "tips": [
      "Encourage the 15 'Rising Stars' to mentor their peers."
    ]
  },
  "radar": {
    "data": [85.0, 72.0, 91.0, 68.0],
    "labels": ["Algorithms", "Database", "Networking", "Math"]
  }
}
GET /api/reports/attendance

Retrieve attendance fatigue metrics and identify students at risk of truancy.

Response Example 
{
  "fatigue": [95.0, 92.0, 85.0, 88.0, 70.0], // Mon-Fri %
  "insights": {
    "status": "Fatigue Detected",
    "summary": "Fridays are showing significant drops in attendance (70%).",
    "tip": "Consider light activities or gamified sessions on Fridays."
  },
  "truancy_prob": [
    {
      "name": "Sarah Lee",
      "perc": 65.0,
      "prob": 35.0
    }
  ]
}
Management API (JSON)

Timetable Management

Operations for updating schedule slots dynamically.

POST /admin/timetable/update_slot

Update or create a timetable slot assignment.

Request Body 
{
  "slot_id": 105,       // Optional (if editing)
  "day": "Monday",
  "period": 3,
  "subject_id": 12,
  "course": "B.Tech",
  "semester": 4
}
Response 
{
  "status": "success"
}
Form Interactions (POST)

Student Operations

Interactions typically handled via HTML Forms. Response is usually a Redirect or HTML render.

POST /student/queries/create

Submit a new query to the faculty/admin.

Form Data Keys 
category: "Academic" | "Hostel" | "Fees" | "Other"
subject: "Linear Algebra Doubt"
description: "Detailed explanation of the issue..."
priority: "Low" | "Medium" | "High"
POST /student/events/register/{event_id}

Register for a university event.

Form Data 
(No body required, ID is in URL)
Form Interactions (POST)

Faculty Operations

Actions for managing academic data.

POST /faculty/attendance

Submit attendance for a class.

Form Data Keys 
course: "B.Tech"
semester: 4
subject_id: 12
date: "2024-03-15"
attendance_{student_id}: "Present" | "Absent" (Repeat for each student)
POST /faculty/marks

Update marks for an exam paper.

Form Data Keys 
exam_paper_id: 45
marks_{student_id}: 85 (Repeat for each student)
